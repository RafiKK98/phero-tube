Scope differences of JavaScript Variable Types (var, let & const)

The scope of a variable in JavaScript determines where the variable can be accessed in the code. There are three different scopes in JavaScript and they are:

  • Global scope: The global scope is the outermost scope in a JavaScript program. Variables declared in the global scope are accessible from anywhere in the program.
  • Function scope: The function scope is the scope of a function. Variables declared in the function scope are only accessible within the function.
  • Block scope: The block scope is the scope of a block of code. Variables declared in the block scope are only accessible within the block.
The keywords var, let, and const are used to declare variables in JavaScript. Variables declared with var are function-scoped. This means that they are accessible from within the function in which they are declared, and from within any nested functions. Variables declared with let are block-scoped. This means that they are only accessible from within the block in which they are declared. Like var, variables declared with const are also block-scoped, but they cannot be reassigned once they are initialized.

JS null and undefined, where to use which

The null and undefined keywords in JavaScript are used to represent the absence of a value. Null is a value that can be assigned to a variable to indicate that the variable has no value. Whereas, Undefined is the default value of a variable that has not been assigned a value. Some use cases of null and undefined are:

  • null is used to indicate that a value is intentionally missing, such as when a user does not provide a value for a form field
  • undefined is used to indicate that a value is missing by default, such as when a variable is declared without being assigned a value.
  • Using the === (triple equals) operator to compare null and undefined to other values will check for both equality and identity, as it will return true if the value on the left side of the operator is equal to the value on the right side and they are the same object.
  • Using the 'typeof' function on null will return an 'Object' whereas undefined will return 'undefined'. Since null is used explicitly to return non-existent values or existent of no value inside a variable, it's of type 'Object'.

What is REST API

REST API stands for Representational State Transfer Application Programming Interface. It is a way of exposing data and functionality over the internet using HTTP requests. REST APIs are designed to be lightweight and easy to use, and they are often used to connect different applications or services. The main principles of REST APIs are:

  • Resource-oriented: REST APIs are based on the concept of resources. A resource is anything that can be identified by a URI (Uniform Resource Identifier). For example, a customer might be a resource, and its URI might be /customers/12345.
  • Stateless: REST APIs are stateless. This means that each request must contain all of the information necessary to process it, and the server does not keep track of the state of the client.
  • Cacheable: REST APIs are designed to be cacheable. This means that the server can return a cached response to a request if the request has not changed.
  • Layered: REST APIs are layered. This means that the client does not need to know how the request is being processed by the server.
  • Uniform Interface: REST APIs use the Uniform Interface (UI) constraints. The UI constraints define how requests and responses are formatted and how errors are handled.
REST APIs are a popular way to expose data and functionality over the internet. They are easy to use, scalable, and reliable. REST APIs are used by a wide variety of applications, including web applications, mobile applications, and cloud-based services. Some examples of REST APIs:
  • The GitHub API allows you to access information about GitHub repositories, users, and organizations.
  • The Google Maps API allows you to get directions, find businesses, and view maps.
  • The Stripe API allows you to process payments.
  • The Twilio API allows you to send and receive text messages.
